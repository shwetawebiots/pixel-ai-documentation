Live API Hardening & Config

When moving to a production environment, security and stability are paramount. Follow these professional best practices to host your Pixel AI Backend.

1

Production Best Practices

  • Process Management: Always use PM2 for continuous monitoring and automatic restarts on failure.
  • Reverse Proxy: Deploy an Nginx or Apache reverse proxy to handle SSL termination and load balancing.
  • Database Security: Restrict MongoDB access via firewall or IP whitelisting. Use MONGODB_SRV for Atlas.
2

Continuous Execution (PM2)

Reliability is key for background AI processing. PM2 ensures your server stays alive 24/7.

# Install process manager globally
npm install pm2 -g

# Start the backend engine
pm2 start server.js --name pixel-ai-api

# Ensure it restarts on system reboot
pm2 save
3

Domain & URL Hardening

Update your core environment variables to match your live production domains.

NODE_ENV=production
PUBLIC_URL=https://api.yourdomain.com
ALLOWED_ORIGINS=https://app.yourdomain.com

CORS Alert: Ensure ALLOWED_ORIGINS exactly matches your frontend URL to prevent browser blocking.
