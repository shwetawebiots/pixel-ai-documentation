User Management

The User Management module empowers administrators to create, update, monitor, and secure user accounts. Features include manual user creation, profile editing, status toggling (active/inactive), and bulk operations for efficient administration.

1. Users List View

The main dashboard displays all registered users in a searchable, filterable table. Columns include avatar, full name, email, role, status (Active/Inactive), registration date, last login, AI credits usage summary, and action buttons. Selecting rows enables bulk actions; clicking a row may open a quick-view sidebar with detailed stats and recent activity.

[IMAGE: Main User Management Dashboard showing user table and activity overview]

2. Add New User

Clicking the “Add User” button opens a modal form. Super Admins can enter full name, email, password (or send invitation link), assign role, set initial subscription plan/credits, and define custom fields if configured. After submission, the user receives a welcome email (if enabled) and appears immediately in the list as Active.

[IMAGE: Add New User Modal with field validation and role selection]

3. Edit User Details

The Edit button opens a detailed form allowing updates to personal info (name, email, avatar), role assignment, status, subscription plan, credit balance, custom metadata, and security notes. Changes are saved instantly with audit logging. Role changes take effect immediately for permissions.

[IMAGE: Detailed User Profile Edit Screen with credit management]

4. Activate / Deactivate User

Admins can quickly toggle a user's status using an inline switch or dedicated button. Deactivating (blocking) prevents login and API access while preserving data. A confirmation modal appears explaining the impact (e.g., “User will be logged out immediately and unable to access services”). Status updates trigger email notifications if configured.

[IMAGE: Confirmation Modal for deactivating a user session]

5. Bulk Delete Users

Select multiple users via checkboxes, then choose “Bulk Delete” from the actions dropdown or toolbar. A guarded confirmation modal warns about permanent data loss (profiles, usage history, generated content — depending on system settings) and may require typing a confirmation phrase. Successful deletion shows a toast notification and refreshes the list.

[IMAGE: Bulk Delete Confirmation Modal with safety warning]

6. Filters, Search & Bulk Toolbar

Use the top search bar for quick keyword lookup (name, email, role). Advanced filters allow narrowing by status, role, registration date, last activity, or credit usage. When rows are selected, a bulk toolbar appears offering actions like Activate, Deactivate, Change Role, Reset Credits, or Delete.

[IMAGE: Advanced search and filtering interface with active chips]

Key Points
  • All destructive actions (status change, delete) require explicit confirmation modals to prevent mistakes.
  • Inactive users retain their data but cannot log in or consume credits.
  • Role changes update permissions instantly across the platform.
  • Bulk operations include summary previews before execution (e.g., “Delete 12 users permanently?”).
  • Every admin action is logged for audit purposes (visible in system logs if enabled).
