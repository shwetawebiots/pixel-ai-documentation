Subscription Plans

The Plan Management module allows administrators to configure subscription plans that control pricing, features, and usage limits across the platform. Admins can create, update, activate or deactivate plans, and define a default plan for new users.

1. All Plans

Displays a complete list of subscription plans in a tabular view with pagination, search, and sorting. Each row shows the plan name, pricing, status, billing cycle, and default indicator, enabling quick comparison and management.

[IMAGE: All Plans List Table with quick search and sorting]

2. Plan Actions

Create Plan: Define plan name, pricing, usage limits, and feature access. Validation ensures required fields and unique identifiers.

Edit Plan: Update existing plan details such as prices, limits, or enabled features. Changes are saved instantly.

Delete Plan: Permanently remove a plan after confirmation to prevent accidental deletion.

View Plan Details: Open a detailed view showing configuration, limits, and enabled features.

[IMAGE: Plan Action buttons and modal interfaces for CRUD operations]

3. Plan Status Management

Active / Inactive Toggle: Enable or disable plans with a confirmation prompt and instant status update.

Default Plan: Mark one plan as the default for new users; only one plan can be set as default at a time.

Status Filters: Quickly filter plans by active or inactive status.

[IMAGE: Status management toolbar and Default plan indicator]

4. Plan Configuration Structure
  • Basic Information: Plan name, slug (auto-generated and unique), and optional description.
  • Pricing Information: Monthly and/or yearly pricing per user with configurable billing cycles.
  • Usage Limits: Limits for group members, broadcast lists, broadcast recipients, daily status posts, total groups, and per-user storage.
  • Feature Flags: Enable or disable core features such as file sharing and video calls per plan.
  • Additional Settings: Display order for sorting, optional trial period duration, default plan flag, and overall plan status.

[IMAGE: Detailed Plan configuration form showing all modular fields]

Key Points
  • All create, update, and delete actions require confirmation to avoid unintended changes.
  • Only one default plan can exist at any time.
  • Validation ensures pricing, limits, and identifiers meet defined constraints.
  • Status changes and updates reflect immediately with clear feedback for administrators.
